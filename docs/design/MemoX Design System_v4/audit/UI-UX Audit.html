MemoX — UI/UX Audit

22 màn hình · 117 states · light + dark · đối chiếu Material 3 + Apple HIG + foundations của chính design system · 06/07/2026.
Nguồn đo đạc: components.css, tokens/*, DOM specs (ui_kits/memox-app/specs/, bounding box px thật) và ảnh render (shots/). Vì DS được sync từ code, mọi fix cần thực hiện ở repo gốc rồi chạy lại /design-sync.

Tóm tắt (tiếng Việt)

Nhìn chung hệ thống rất tốt: phủ state đầy đủ (loading / empty / error cho hầu hết flow), copy chuẩn giọng "coach", feedback đúng/sai dùng cả màu + icon, dialog xoá có xác nhận đúng chuẩn. Các vấn đề tìm thấy tập trung vào 4 nhóm hệ thống — sửa 1 lần ở token/component sẽ tự lành trên cả 22 màn hình:

  • MAJORContrast (WCAG): text-tertiary light chỉ đạt 2.4:1 nhưng đang dùng cho label 11–13px ở 19/22 màn hình; token on-accent/on-info dark bị hỏng (1.34:1); warning/success/error đặt trực tiếp trên nền trắng đều rớt AA. → Sửa ~6 giá trị token (mục G1).
  • MAJORTouch target: 8 control dùng chung nhỏ hơn 44px (chip 34, switch 32, btn--sm 38, icon-btn--sm 36, segmented 38, "See all" 20px…), xuất hiện ở 36+ vị trí. Đặc biệt nút Save của Card Editor — hành động chính của màn hình — chỉ cao 38px. → Mở rộng vùng chạm bằng pseudo-element, giữ nguyên visual (mục G2).
  • MAJORNhất quán bố cục: 4 kiểu hiển thị tiến độ khác nhau (Review: bar + "7/20" ✓; Study session: chỉ chữ "16%"; Games: bar không số; Player: chấm tròn); 3 kiểu xếp cặp nút hành động cuối màn; nút "Leave" (huỷ phiên học) đang mang màu primary thay vì danger — ngược với dialog xoá thẻ. → Chuẩn hoá 3 convention (mục G3–G5).
  • MINORChi tiết màn hình: FAB che nội dung cuối danh sách (padding đáy 96px < mức cần ~148px); biểu đồ Stats không có nhãn trục/chú giải; chữ 11px ở nav + badge; dấu "✓" unicode trong badge thay vì icon. (G6–G8).

Thứ tự ưu tiên đề xuất: ① token contrast (G1) → ② hit-area component (G2) → ③ convention tiến độ + action bar + destructive (G3–G5) → ④ FAB clearance, chart labels, chi tiết từng màn.

G3Progress indication — 4 different patterns for the same concept

  • GOODReview: linear bar + "7/20" count — the best pattern; make it the standard.
  • MAJORStudy Session: text "16%" only (study-session/progress is a 230×20 text div) — no visual bar, no card count, and % is harder to reason about mid-session.
  • MINORGames (matching/mc/typing/recall): bar with no count.
  • MINORPlayer: 8 dots — breaks down beyond ~12 cards.

Fix: one shared progress header = linear ProgressBar + "n / N" label, used by session, games and player alike.

G4Bottom paired-action layouts — 3 competing conventions

  • Study session: full-width primary Next ✓ (single action)
  • Due-review & Fill: ghost left + primary right, unequal widths, right-aligned
  • Recall: Forgot (ghost) + Got it (outline) — neither is primary although this is the core grading interaction of the whole app

Fix: define one action-bar convention (e.g. equal-width pair, 48px+, primary on the right; single action = full-width). Recall's grade buttons deserve the strongest visual weight in the flow.

G5Destructive-action conventions are split

  • GOODDelete card / delete deck dialogs: ghost Cancel + danger Delete — correct.
  • MAJORStudy-session exit dialog: "Leave" (discards stage progress) is styled primary brand — the visual language says "recommended action". Should be danger (or at minimum outline, with "Stay" as the emphasized safe action).
  • MINORAccount "Sign out": full-width solid danger is the loudest element on an otherwise calm screen; HIG favors a quieter destructive row/outline + confirm dialog.

G6FAB occludes end-of-list content

.app__body bottom padding = nav 72 + 24 = 96px, but the FAB band (60px + margin) sits above the nav and overlaps the last row's trailing badge/progress on dashboard, library and deck-detail. Fix: bottom padding ≥ nav + FAB + 16 ≈ 148px on FAB screens.

G7Type scale floor

11px is in use for badge text and bottom-nav labels (with text-tertiary idle color — double penalty with G1). M3 label-small floor is 11sp but with 4.5:1 contrast; recommend 12px minimum + fix idle color to text-secondary.

G8Smaller systemic details

  • NITDeckRow renders a unicode "✓" inside MxBadge when due=0 — the system's rule is icons, not unicode glyphs; use check Material symbol or a "Done" soft badge.
  • NITStat uses --memox-icon-size-md (22px) as a font size — token category misuse; add/use a font-size token.
  • NITSheet grab-handle uses divider (8% alpha) — nearly invisible; use border-strong.
  • MINORFocus rings exist (3px ✓) but chips/switch rely on default only — verify :focus-visible on every interactive primitive.

Per-screen findings

Dashboard — Today

5 states: loaded · empty · loading · goal-met · streak-reset

Hero tốt, số liệu nổi bật đúng chuẩn "numbers lead"; lỗi chính là FAB che hàng deck cuối, "See all" quá nhỏ và caption goal khó đọc.

  • MAJOR"Review" FAB overlaps the last deck row's badge + progress bar → G6.
  • MAJOR"See all" section action is a 41×20px bare-text hit area → G2.
  • MINORAppbar avatar (→ profile) is 32px tappable → G2.
  • MINORDaily-goal caption "Met when minutes OR words reached" — 13px text-tertiary (2.4:1, G1) and reads like spec text; suggest "Counts minutes or words — whichever hits first", in text-secondary.
  • MINOREmpty state: below the TODAY card the lower two-thirds is blank; the "start your streak" callout floats at top. Consider an EmptyState block (icon + line + "Create a deck") in the void.
  • NITStreak tile is primary-soft while mastered tile is muted — if intentional emphasis, document it; it reads as accidental.
  • GOODLoading skeleton mirrors real layout; goal-met and streak-reset states are covered with the right tones.

Library

10 states incl. loaded · empty · loading · error · search-active · sheets/menus

Cấu trúc list tốt, state phủ đủ; điểm trừ: search chỉ là icon trần (không nhất quán với Deck Detail), FAB che hàng cuối.

  • MINORSearch entry is a bare icon while Deck Detail uses a full search dock — two affordances for the same action one level apart. Suggest the dock (or M3 search bar) here too; it also fixes the small icon target.
  • MAJOR"New" FAB overlaps the last deck row → G6; search-clear 36px → G2.
  • MINORPlay-sheet title (deck name) is 13px CAPS text-tertiary → G1; grab handle nearly invisible → G8.
  • NITLanguage-pair pill sits between search and sort with no grouping cue; consider anchoring it to the app bar (it changes global scope, not list filtering).
  • GOODEmpty state ("Create deck" + "Add words") and error state with Retry are exemplary; due badges use soft tones consistently.

Deck Detail

13 states — the deepest coverage in the kit

Màn hình chuẩn mực nhất về state; lỗi chủ yếu là target nhỏ (sort/clear/chip) và label section chìm.

  • MAJORSort 36px, search-clear 36px, filter chips 34px → G2.
  • MINOR"SUB-DECKS" / "CARDS" section labels: 13px CAPS text-tertiary → G1.
  • MINOR"Add" FAB overlaps last card row → G6.
  • NITMastered sub-deck shows unicode "✓" badge → G8.
  • GOODDelete-confirm dialog (ghost Cancel + danger Delete) is the correct destructive pattern — promote it as the app-wide convention (see G5). Search, filters, menus, empty, error, loading all present.

Search

5 states: empty-recent · results · no-results · …

Pattern tốt (recent → results → no-results); chỉ cần nâng target icon và contrast dòng meta.

  • MAJORRecent-fill (north_west) and clear buttons 36px → G2.
  • MINORResult meta line (deck name) 13px text-tertiary → G1; "RECENT" label likewise.
  • NITHidden-card result at 55% opacity stacks on already-low-contrast meta — consider a "Hidden" soft badge instead of dimming the whole row.
  • GOODNo-results copy quotes the query and suggests recovery — textbook error copy.

Card Editor

6 states: create · edit · validation · multi-meaning · audio · duplicate

Form tốt, validation rõ; lỗi lớn nhất: nút Save — hành động chính — chỉ 38px và đặt kiểu app-bar nhỏ.

  • MAJORSave is a 38px btn--sm → G2. As the screen's primary commit action, consider a full-width bottom Save (keeps thumb reach) or at least a 48px bar button.
  • MINORGender chips 34px, Hide-card switch 32px → G2.
  • MINORError text uses raw error on canvas (4.12:1) → G1; required fields marked by red "*" only — add "(required)" to labels or rely on inline errors for screen readers.
  • NITSecondary-meaning field (multi-meaning state) shows no remove affordance — verify one exists.
  • GOODDisabled Save in create state, red outline + message pattern, and audio row are all clear.

Study Session (5 stages)

10 states incl. stage1–5 · due-review · relearn · exit · resume-error · answer-save-error

Flow lõi của app: thiếu progress bar trực quan (chỉ "16%"), dialog thoát dùng nhầm màu primary cho hành động phá huỷ.

  • MAJORProgress is a bare "16%" text — no linear bar, no card count → G3. Adopt Review's bar + "n/N".
  • MAJORExit dialog: "Leave" styled primary while it discards progress → G5.
  • MINORRelearn/Next pair: unequal, right-aligned → G4.
  • NITStage 1 card centers content with large fixed blank zones; on smaller frames (SE-class, 667pt) verify the Next button stays above the fold.
  • GOODDue-review warning note, relearn "not counted" note, and both error states (resume, answer-save) are exactly the resilience states most apps forget.

Review

4 states: browsing · editing · audio · end

Màn hình có progress chuẩn nhất — nên lấy làm mẫu; chỉnh target nút edit/Cancel/Save và dòng "Swipe to continue".

  • GOODLinear bar + "7/20" — the reference progress pattern (G3).
  • MAJOREdit pencil 36px; editing-state Cancel/Save 38px → G2.
  • MINOR"Swipe to continue" + chevrons: 13px tertiary → G1; chevrons look tappable but are tiny — either make them real 48px paging buttons or drop them.
  • NIT"MEANING" card overline is tertiary CAPS → G1.

Study Result

7 states incl. standard · goal-met/missed · many-wrong · finalizing · finalize-error · retry

  • GOODStat tiles + streak card + single primary CTA — clean hierarchy; finalize-error with Retry / Not now is a great resilience state.
  • NITStat labels 12px text-secondary — fine after the G1 alpha bump; keep ≥12px.
  • NITStat number font-size borrows an icon-size token → G8.

Game Picker

Match

6 states: playing · selected · correct · wrong · almost · complete

Grid ghép cặp gọn; thiếu số đếm tiến độ và một dòng hướng dẫn ngắn.

  • MINORProgress bar has no count/label → G3 ("3/5 pairs").
  • NITNo instruction line for first-time players ("Tap a term, then its match") — one 13px secondary line under the bar suffices.
  • NITVerify wrong-pair feedback pairs color with a non-color cue (icon/shake) like MC does — the wrong state should not be border-color-only.
  • GOODRound-complete state is warm and offers Next round immediately.

Guess

4 states: waiting · correct · wrong · complete

Feedback đúng/sai mẫu mực (màu + icon); chỉ cần nâng nút edit và thêm số đếm.

Recall

5 states: before-reveal · revealed · remembered · forgot · complete

Đây là tương tác chấm điểm cốt lõi nhưng cặp nút Forgot / Got it lại yếu nhất về thị giác — cần nâng cấp thành cặp nút lớn, cân bằng.

  • MAJORGrade actions "Forgot" (ghost) / "Got it" (outline): neither is primary, widths unequal, right-aligned → G4. Recommend equal-width 48px+ pair — e.g. outline-danger Forgot / filled-success Got it — full-width row.
  • MINOREdit icon 36px → G2; uncounted progress → G3.
  • GOODReveal flow (question card → answer card) reads clearly.

Fill

6 states: waiting · typing · hint · correct · wrong · complete

Flow gõ đáp án ổn, có hint; chuẩn hoá lại hàng nút Help/Check theo convention chung.

  • MINORHelp (ghost) + Check (primary): unequal, right-aligned pair → G4; with the keyboard up, Check should hug the keyboard full-width.

Import

5 states: source · mapping · preview · dup-warning · done

Wizard 3 bước rõ ràng, có cảnh báo trùng; vài target nhỏ và copy cảnh báo nên đổi từ câu hỏi sang khẳng định.

  • MINORSeparator chips 34px; column-mapping expand buttons 36px → G2.
  • NITDup-warning note asks "import anyway?" but carries no action — the answer lives in the main CTA below. Reword declaratively: "8 cards already exist — they'll be skipped/overwritten."

Export

Stats

4 states: loaded · loading · insufficient · scope-switch

Biểu đồ đẹp nhưng "câm": heatmap không có nhãn thứ/chú giải đậm–nhạt, bar chart không có nhãn ngày/giá trị.

  • MAJORStudy-calendar heatmap: no weekday labels and no less→more legend — intensity is uninterpretable; add M3-style axis + legend.
  • MAJORTime-per-week bars: no day labels or values — add x-axis labels and value-on-tap (or max value marker).
  • MINORSegmented segs 38px → G2.
  • NITStreak tiles reuse the dashboard's mixed tones (primary-soft vs muted) — same intentionality question.
  • GOODInsufficient-data state explains what will appear and how to get it.

Settings

3 states: loaded · group-expanded · about

Chuẩn list-settings, phân nhóm tốt; chỉnh contrast label nhóm và switch target.

  • MINOR"STUDYING" (and sibling) group labels: 13px CAPS tertiary → G1; switches in expanded groups 32px → G2.
  • NITGame-settings row shows a bare trailing "5" next to the chevron — cryptic; the subtitle already says "5 words/round", drop the trailing value.
  • GOODRow anatomy (icon tile · title · current-value subtitle · chevron) is consistent and scannable throughout.

Account & Sync

5 states: signed-out · signed-in · syncing · conflict · offline

State máy đồng bộ phủ tốt; nút Sign out đỏ đặc full-width quá "gào" so với phần còn lại, khoảng trống dưới lớn.

  • MINORFull-width solid-danger "Sign out" → G5; white-on-error is 4.12:1 → G1. An outline-danger row + confirm dialog is calmer and safer.
  • MINOR"Sync now" 38px → G2.
  • NITThe "conflict" state actually shows the resolution ("Merged · last-write-wins") — consider a pre-resolution state if users can choose.
  • NITLower half of the screen is empty — sync history or storage info could ground it (ask product first).

Reminders

3 states: on · off · time-picker

Số giờ 13:00 làm hero — đúng tinh thần; nâng chip ngày + switch, kiểm tra độ phân biệt chip chọn/không chọn.

  • MINORDay chips 34px, master switch 32px → G2.
  • MINORSelected vs unselected day chips are hard to tell apart at a glance (soft tint vs white + hairline) — add the emphasis border or a check to selected chips.
  • NIT"REMINDER TIME" / "REPEAT" labels tertiary CAPS → G1; time-edit icon appears <44px — verify the whole time card opens the picker.

Theme

3 states: light · dark · accent-picked

Card preview trực tiếp là điểm sáng; swatch màu cần label/aria và target 48px.

  • MINORAccent swatches are 40px, color-only, unlabeled — extend to 48px targets and give each an accessible name (Indigo, Violet, Green…) → G2.
  • NITSegmented segs 38px → G2.
  • GOODLive preview card + text-size control — exactly the right anatomy for a theme screen.

Drawer & Languages

3 states: open · add-language · remove-language

Menu drawer gọn, activity tóm tắt trên đầu hợp lý; kiểm tra bề rộng drawer và contrast label.

  • MINORDrawer spans ~90% of the frame — M3 modal drawer max is 360dp with a visible scrim edge; narrow it so the scrim (and tap-to-close) is clearly available.
  • NIT"TODAY'S ACTIVITY" label tertiary CAPS → G1.
  • GOODAdd/remove-language flows have dedicated states; icon + label rows are comfortable.
